# ParkLog — CLAUDE.md

## Project Overview
ParkLog (דרכלוג) — Parking lot vehicle entry management system.
Two interfaces:
- **VaultEntry** — Mobile-first data entry form for employees
- **CommandCenter** — Admin dashboard with KPIs, charts, filters, exports

## Tech Stack
- HTML5 Semantic (`<html lang="es" dir="ltr">`)
- CSS3 — NO frameworks. Flexbox + Grid, CSS Variables only
- Vanilla JavaScript ES6+ — NO jQuery or libraries
- Chart.js (CDN) — only in CommandCenter
- Google Sheets as backend storage
- **Google Apps Script as Web App** (doGet/doPost) — serves as backend proxy
- Frontend communicates ONLY with Apps Script URL — NEVER directly with Sheets API

## Architecture — Frontend ↔ Backend
```
Browser (frontend)          Google Cloud (backend)
┌──────────────┐            ┌──────────────────┐         ┌──────────────┐
│ vault-entry  │            │  Google Apps      │         │              │
│ commandcenter│ ── fetch ──│  Script Web App   │── r/w ──│ Google Sheet │
│ sheets.js    │            │  (doGet/doPost)   │         │              │
└──────────────┘            └──────────────────┘         └──────────────┘
   No API keys!              Keys stay here               Data storage
```
- **sheets.js** — abstraction layer, sends fetch() to Apps Script URL
- **Apps Script** — handles all Sheets operations server-side, validates input
- **No API keys in frontend** — Apps Script has native access to Sheets

## Data Model (Google Sheets)
### Sheet 1: Vehicles (one row per unique vehicle)
| Field | Type | Required | Notes |
|-------|------|----------|-------|
| vehicle_id | String | Yes | UUID — generated by Apps Script |
| placa | String | Yes | License plate, uppercase, unique, 2-10 chars, A-Z 0-9 dash |
| tipo | String | Yes | `auto` (default) or `moto` — only two values |
| first_seen | Date | Yes | Set once on creation — never changes |
| last_seen | Date | Yes | Updated on every entry |
| total_visits | Number | Yes | Incremented on every entry |
| notes | String | No | Max 300 chars, validated server-side |
| created_by | String | No | "anonymous" in Phase 1, user ID in Phase 2 |

### Sheet 2: Entries (one row per entry event)
| Field | Type | Required | Notes |
|-------|------|----------|-------|
| entry_id | String | Yes | UUID — generated by Apps Script |
| vehicle_id | String | Yes | FK to Vehicles (denormalized for query speed) |
| placa | String | Yes | Plate number (denormalized for query speed) |
| entry_date | Date | Yes | Set by Apps Script server-side |
| entry_time | Time | Yes | Set by Apps Script server-side |
| notes_entry | String | No | Max 300 chars, validated server-side |
| created_by | String | No | "anonymous" in Phase 1, user ID in Phase 2 |

### New Vehicle Detection Logic
A vehicle is "new" when it does NOT exist in the Vehicles sheet at the moment of entry.
- On entry: search Vehicles sheet for placa
- NOT found → **New vehicle** (🟢 green badge) — add to Vehicles, add to Entries
- Found → **Known vehicle** (🔵 blue badge) — update last_seen + total_visits, add to Entries
- No boolean flags needed — "newness" is determined by existence check at entry time
- On save: Apps Script creates entry + creates/updates vehicle **atomically**

### CommandCenter: "New vehicles" reports
To query "new vehicles on date X": filter Vehicles where `first_seen === X`
This works because first_seen is set once and never changes.

### Update Logic (on every entry for a known vehicle)
1. `last_seen` → set to current date
2. `total_visits` → increment by 1
3. Both updates happen in the same Apps Script call as entry creation

### Placa Validation Rules
- Characters: A-Z, 0-9, dash only
- Length: 2-10 characters
- Auto-uppercase on every keystroke (frontend)
- Same validation duplicated in Apps Script (server-side)
- No spaces allowed

## Security Rules — IRON RULE
- **NO API keys in frontend code** — NEVER
- Apps Script Web App URL stored in `config.js` (not secret — URL alone cannot bypass Apps Script validation)
- Apps Script validates Origin header against allowlist
- Apps Script validates all input server-side (placa format, notes length, etc.)
- All user text rendered with `textContent` — NEVER `innerHTML` (XSS prevention)
- GitHub repo MUST be private

## Error Handling & Network Resilience
### Network Failures (VaultEntry)
- On save failure: red toast "No hay conexión — Reintentando / אין חיבור — מנסה שוב"
- Entry queued in localStorage, auto-retry when online
- Offline indicator in header when `navigator.onLine === false`
- Queued entries sent in order with original timestamps on reconnection

### Network Failures (CommandCenter)
- Data cached for 60 seconds — show "last updated" timestamp
- On load failure: show last cached data + warning banner
- Max 1 API call per second (client-side rate limiting)

### Vehicle Lookup Failure
- If plate search fails due to network: show form without badge
- Allow save — entry logged without new/known status
- Status resolved on next successful sync

### Double Submission Prevention
- Submit button disabled for 3 seconds after click
- Form clears after successful save
- Confirmation banner shows saved plate number
- Loading spinner on button during API call

## CommandCenter Definitions
### "Week" Definition
- ISO 8601: Monday 00:00 to Sunday 23:59
- KPI "weekly entries" = current week (Monday to now)
- Bar chart: last 8 complete weeks + current partial week

### "New vehicles on date X"
- Filter: `Vehicles.first_seen === X`
- Doughnut chart: new (first_seen = today) vs known (first_seen < today)

### Note Editing
- Pencil icon on each row — opens modal with textarea
- Can edit: Vehicle notes (Vehicles sheet) and Entry notes (Entries sheet)
- Save writes directly to respective sheet via Apps Script
- No concurrent edit protection in Phase 1 (last write wins)

### Exports
| # | Export | Method | Output |
|---|--------|--------|--------|
| 1 ⭐ | New vehicles today | Big button + "Copy to clipboard" | Simple plate list / CSV |
| 2 | New vehicles by date X | Date picker + export | CSV |
| 3 | Vehicles by last_seen range | Date range picker + export | CSV |
| 4 | All data | "Export all" button | Full CSV |

## Loading & Empty States
### Loading
- Skeleton loaders (gray shapes matching content layout) with shimmer animation
- VaultEntry: spinner overlay on plate field during lookup
- CommandCenter: skeleton for KPI cards, charts, and table rows

### Empty States

## Design Principles

### 1. Whitespace & Layout
- Generous whitespace between sections — never feel cramped
- Card-based layout with subtle shadows (--shadow-sm)
- Rounded corners (--radius-md to --radius-lg) for friendly feel
- Single-column layout on mobile, grid on desktop

### 2. Typography
- **Inter** for body text (excellent readability, all Latin + Hebrew chars)
- **Plus Jakarta Sans** for headings (modern, friendly)
- Body: 16px min on mobile (never smaller)
- Line-height: 1.5 for body, 1.2 for headings
- Font weights: 400 regular, 500 medium, 600 semibold, 700 bold

### 3. High Contrast
- All text must pass WCAG AA (4.5:1 ratio minimum)
- Primary text (#1E293B) on white = 12.6:1 ratio
- Never rely on color alone — always pair with icons/text
- Test in grayscale to verify readability

### 4. Icons
- Use **Lucide Icons** (CDN) as primary icon library
- Clean, consistent 24px stroke style
- Use icons next to labels, not as replacements
- Emoji for vehicle types: 🚗 auto, 🛵 moto
- Status badges: 🟢 new vehicle (green), 🔵 known vehicle (blue)

### 5. Buttons & Interactive Elements
- Minimum touch target: 44x44px (Apple HIG)
- Clear hover/focus/active states with transitions
- Primary buttons: solid --color-primary with white text
- Secondary buttons: outlined with --color-primary border
- Disabled state: reduced opacity (0.5) + no pointer

### 6. Forms (VaultEntry)
- Single-column layout always
- Labels ABOVE inputs (never placeholder-only)
- Input height: 48px minimum on mobile
- Inline validation on blur (not on every keystroke)
- Error messages: red text + icon, below the field
- Success feedback: green confirmation with checkmark
- **New vehicle badge**: 🟢 bright green badge, immediately visible after plate lookup
- **Known vehicle badge**: 🔵 blue badge with last seen date
- **Session new vehicles list**: after each new vehicle entry, a running list
  of all new vehicles entered in the current session appears below the form.
  Each item shows: plate number + entry time. List is stored in sessionStorage.
  Includes a "copy all plates" button for quick export.

### 7. Tables (CommandCenter)
- Sticky header on scroll
- Zebra striping: alternate rows with #F8FAFC
- Fixed first column (placa) on horizontal scroll
- Mobile: convert to card layout or collapsible rows
- Sort indicators: arrow icons on column headers

### 8. Animations & Transitions
- Subtle, purposeful microinteractions only
- Transition duration: 150-200ms for UI elements
- Ease function: cubic-bezier(0.4, 0, 0.2, 1)
- Badge appearance: fade-in with slight scale
- No distracting animations — keep it professional

### 9. Loading States
- Skeleton loading (shape of content) instead of spinners
- Shimmer animation for loading placeholders
- Disable submit button during API calls with loading indicator

### 10. Responsive Breakpoints
```css
/* Mobile-first approach */
/* Base: < 480px — phone */
/* sm: >= 480px — large phone */
/* md: >= 768px — tablet */
/* lg: >= 1024px — desktop */
```
- **< 480px**: VaultEntry single-column, CommandCenter table → card layout, column toggle essential
- **>= 768px**: CommandCenter table with horizontal scroll, charts in 1-column grid
- **>= 1024px**: Full CommandCenter layout, 2-column chart grid, all table columns visible

### 11. Accessibility
- All interactive elements keyboard-navigable
- Focus rings visible (--color-primary outline)
- aria-labels on icon-only buttons
- Role attributes on dynamic content
- Screen reader friendly status messages
